## Critical Gaps

### GAP-01: `highlight.define_groups()` never called — cursors are invisible at runtime

**Requirement:** FEAT-07
**Severity:** Critical — user-visible behavior broken

`highlight.define_groups()` defines the four VM_ highlight groups that all extmark rendering depends on. It is implemented in `lua/visual-multi/highlight.lua` (line 23) and tested in specs, but has no call site in `init.lua`, `setup()`, or `session.start()`.

At runtime, when `hl.redraw(session)` fires:
- `nvim_buf_set_extmark(..., { hl_group = 'VM_Cursor', ... })` is called
- `VM_Cursor` group was never defined → Neovim silently ignores the highlight
- Cursor positions are invisible to the user

**Fix:** Add `require('visual-multi.highlight').define_groups()` to `M.setup()` in `init.lua`, or to `session.start()` before first `redraw()` is called.

**Evidence:** No call to `define_groups` in `lua/visual-multi/init.lua` or any other production module (confirmed by integration checker grep).

### GAP-02: `edit.g_increment` missing `undojoin` — multi-cursor g<C-a>/g<C-x> produces N undo entries

**Requirements:** FEAT-06, FEAT-10
**Severity:** Significant — FEAT-06 contract violated for g_increment operation

`M.exec` correctly calls `vim.cmd('undojoin')` between cursor iterations (lines 91-94), merging per-cursor feedkeys edits into a single undo entry. `M.g_increment` does not:

```lua
-- edit.lua (g_increment loop) — missing undojoin:
for step, idx in ipairs(order) do
  ...
  pcall(vim.api.nvim_feedkeys, encoded, 'x', false)
  -- NO undojoin here: each cursor's <C-a> repetitions are separate undo entries
end
```

With N cursors, pressing `u` after `g<C-a>` undoes only the last cursor's increments. The user must press `u` N times to fully revert.

The unit tests for this passed because they tested only single-cursor scenarios (`g_increment creates exactly one undo entry` — but the test session had only 3 cursors and the test was checking `after - before == 1` using the undolevels flush trick, which resets the baseline — actual multi-cursor undo behavior was not validated).

**Fix:** Add `pcall(vim.cmd, 'undojoin')` after the first cursor iteration in `g_increment`'s loop (same pattern as `exec` at lines 91-94).
