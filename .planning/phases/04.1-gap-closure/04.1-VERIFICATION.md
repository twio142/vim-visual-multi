### Summary

Both gaps identified in the v1.0 milestone audit are fully closed:

**GAP-01 (define_groups never called from M.setup):** Fixed by adding `require('visual-multi.highlight').define_groups()` as line 18 of `init.lua`, immediately after `config.apply(opts)`. The GAP-01 regression spec in `highlight_spec.lua` (T_hl['setup() defines VM_ highlight groups']) calls `setup()` and uses `nvim_get_hl(0, {name='VM_Cursor'})` to confirm the group resolves to a non-empty table. This satisfies FEAT-07.

**GAP-02 (g_increment produces N undo entries for N cursors):** Fixed by adding the `first/undojoin` pattern to the g_increment loop in `edit.lua`, mirroring the existing pattern in M.exec. The GAP-02 regression spec in `edit_spec.lua` (T['g_increment multi-cursor produces exactly one undo entry (GAP-02 regression)']) uses a 3-cursor file-backed session with proper undolevels=-1 baseline flush and asserts `after - before == 1`. This satisfies FEAT-06 and FEAT-10.

The test count increased from 100 to 102 (2 new regression specs). All 102 tests pass.
