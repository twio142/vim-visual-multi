**1. `_col_end(buf, row, col)` — safe byte-width helper (local)**

Returns `col + byte_width_of_char_at(buf, row, col)`. Handles: empty lines, EOL,
multibyte chars (CJK = 3 bytes, accented = 2 bytes, ASCII = 1 byte). PITFALL-14:
`col + 1` is wrong for multibyte — always use this helper for single-char extmarks.

```lua
local function _col_end(buf, row, col)
  local line = vim.api.nvim_buf_get_lines(buf, row, row + 1, false)[1]
  if not line or #line == 0 or col >= #line then
    return col + 1  -- EOL or empty: strict=false handles out-of-bounds
  end
  local char = vim.fn.matchstr(line, '.', col)  -- char at byte offset col (multibyte-aware)
  local bw = #char
  return col + (bw > 0 and bw or 1)
end
```

**2. `_draw_cursor_region(region, is_primary)` (local)**

Single-char highlight using VM_Cursor (primary) or VM_CursorSecondary (others).
Reads current position from sel_mark_id via nvim_buf_get_extmark_by_id.
After `nvim_buf_clear_namespace`, stored IDs are stale — create new marks (no `id=`)
and store the returned ID back on the region. Priority 200, hl_mode 'combine',
strict false, right_gravity true.

```lua
local function _draw_cursor_region(region, is_primary)
  local hl_group = is_primary and 'VM_Cursor' or 'VM_CursorSecondary'
  local info = vim.api.nvim_buf_get_extmark_by_id(region.buf, M.ns, region.sel_mark_id, {})
  local row, col = info[1], info[2]
  region.sel_mark_id = vim.api.nvim_buf_set_extmark(region.buf, M.ns, row, col, {
    end_row       = row,
    end_col       = _col_end(region.buf, row, col),
    hl_group      = hl_group,
    priority      = 200,
    hl_mode       = 'combine',
    strict        = false,
    right_gravity = true,
  })
  -- tip_mark_id not used in cursor mode
  region.tip_mark_id = nil
end
```

CRITICAL: Do NOT pass `id = region.sel_mark_id` — after clear_namespace the old ID
is gone; passing a stale id= creates a new mark with a server-assigned ID anyway
(Neovim behavior) but is semantically wrong and makes reasoning harder. Always create
new and store back.

**3. `_draw_extend_region(region, is_primary)` (local)**

Two extmarks per region:
- Selection span: anchor → cursor tip, using VM_Extend/VM_ExtendSecondary, priority 200
- Cursor-tip overlay: single char at cursor tip, using VM_Cursor/VM_CursorSecondary, priority 201

Zero-width fallback: if anchor pos == tip pos (same row and col), call
`_draw_cursor_region(region, is_primary)` and return immediately (renders as cursor, not invisible).

Read tip position from sel_mark_id. Read anchor position from anchor_mark_id.
If anchor_mark_id is nil (shouldn't happen in extend mode but guard anyway),
fall back to cursor mode render.

Direction: selection span always goes from min(anchor, tip) to max(anchor, tip)
(compare row first, then col). The `end_col` of the selection extmark should cover
the last character fully: use `_col_end` on the end position.

After the clear-namespace in redraw, both sel_mark_id and anchor_mark_id are stale.
Create new marks (no `id=`) and store returned IDs back. For anchor_mark_id, create
a new invisible tracking mark with `right_gravity = false` (no hl_group).

```lua
local function _draw_extend_region(region, is_primary)
  local sel_hl = is_primary and 'VM_Extend'  or 'VM_ExtendSecondary'
  local tip_hl = is_primary and 'VM_Cursor'  or 'VM_CursorSecondary'

  -- Read tip position (extmark auto-tracked positions survive clear-namespace?
  -- NO — after clear_namespace the mark is GONE. We must read it BEFORE the clear.
  -- NOTE: redraw() calls clear THEN calls this function, so at this point sel_mark_id
  -- is the ID from the PREVIOUS redraw, already cleared. The position we stored is
  -- now gone from the namespace.
  --
  -- RESOLUTION (from RESEARCH.md open question 1): redraw() must NOT clear before
  -- reading positions. The correct order is:
  --   1. Read all positions from current extmarks
  --   2. Clear namespace
  --   3. Write all new extmarks
  -- This is handled in redraw() below — _draw_extend_region receives pre-read positions.
  -- See the redraw() implementation for the actual call pattern.
  --
  -- This function receives: region with .tip_row, .tip_col, .anc_row, .anc_col
  -- set by redraw() before the clear. Do NOT call nvim_buf_get_extmark_by_id here.

  local tip_row, tip_col = region._tip_row, region._tip_col
  local anc_row, anc_col = region._anc_row, region._anc_col

  -- Zero-width fallback: anchor == tip → render as cursor
  if tip_row == anc_row and tip_col == anc_col then
    _draw_cursor_region(region, is_primary)
    return
  end

  -- Selection span: min(anchor, tip) → max(anchor, tip)
  local start_row, start_col, end_row, end_col
  if anc_row < tip_row or (anc_row == tip_row and anc_col <= tip_col) then
    start_row, start_col = anc_row, anc_col
    end_row,   end_col   = tip_row, _col_end(region.buf, tip_row, tip_col)
  else
    start_row, start_col = tip_row, tip_col
    end_row,   end_col   = anc_row, _col_end(region.buf, anc_row, anc_col)
  end

  region.sel_mark_id = vim.api.nvim_buf_set_extmark(region.buf, M.ns,
    start_row, start_col, {
      end_row  = end_row,
      end_col  = end_col,
      hl_group = sel_hl,
      priority = 200,
      hl_mode  = 'combine',
      strict   = false,
    })

  -- Cursor-tip overlay (higher priority so it appears on top of selection)
  region.tip_mark_id = vim.api.nvim_buf_set_extmark(region.buf, M.ns,
    tip_row, tip_col, {
      end_row  = tip_row,
      end_col  = _col_end(region.buf, tip_row, tip_col),
      hl_group = tip_hl,
      priority = 201,
      hl_mode  = 'combine',
      strict   = false,
    })

  -- Recreate anchor tracking mark (right_gravity=false: doesn't drift on insert-before)
  region.anchor_mark_id = vim.api.nvim_buf_set_extmark(region.buf, M.ns,
    anc_row, anc_col, {
      right_gravity = false,
      strict        = false,
    })
end
```

**4. `M.redraw(session)` (exported)**

Read-positions-first, then clear, then draw. This is the correct order to avoid
reading stale IDs after clear_namespace.

```lua
function M.redraw(session)
  if session._stopped then return end
  if #session.cursors == 0 then return end

  -- Phase 1: Read all current positions BEFORE clearing (marks are still valid)
  for _, region in ipairs(session.cursors) do
    if not region._stopped then
      local tip_info = vim.api.nvim_buf_get_extmark_by_id(
        region.buf, M.ns, region.sel_mark_id, {})
      region._tip_row = tip_info[1]
      region._tip_col = tip_info[2]
      if session.extend_mode and region.anchor_mark_id then
        local anc_info = vim.api.nvim_buf_get_extmark_by_id(
          region.buf, M.ns, region.anchor_mark_id, {})
        region._anc_row = anc_info[1]
        region._anc_col = anc_info[2]
      end
    end
  end

  -- Phase 2: Atomic clear (one call; no ghost marks possible)
  vim.api.nvim_buf_clear_namespace(session.buf, M.ns, 0, -1)

  -- Phase 3: Redraw each region with correct primary/secondary group
  local primary_idx = session.primary_idx
  for i, region in ipairs(session.cursors) do
    if not region._stopped then
      local is_primary = (i == primary_idx)
      if session.extend_mode and region.mode == 'extend' then
        _draw_extend_region(region, is_primary)
      else
        _draw_cursor_region(region, is_primary)
      end
    end
  end
end
```

**session.lua — expand toggle_mode():**

Replace the Phase 2 stub toggle_mode with the Phase 3 version that:
1. Sets anchor on Cursor→Extend transition (for each cursor, reads current pos from
   sel_mark_id, creates anchor extmark via Region.new's anchor pattern — but since
   region objects already exist, just update region fields directly and create
   anchor_mark_id extmark in-place).
2. Collapses anchor on Extend→Cursor transition (clears anchor, clears tip_mark_id
   reference, sets mode = 'cursor').
3. Calls `require('visual-multi.highlight').redraw(session)` after mode flip.

```lua
function M.toggle_mode(session)
  local hl = require('visual-multi.highlight')
  if session.extend_mode then
    -- Extend → Cursor: collapse each selection to its cursor-tip position
    for _, r in ipairs(session.cursors) do
      if not r._stopped then
        -- Delete anchor tracking mark (will be recreated by redraw if needed)
        if r.anchor_mark_id then
          pcall(vim.api.nvim_buf_del_extmark, r.buf, hl.ns, r.anchor_mark_id)
          r.anchor_mark_id = nil
        end
        r.tip_mark_id = nil   -- will be gone after next clear-all redraw
        r.mode = 'cursor'
      end
    end
    session.extend_mode = false
  else
    -- Cursor → Extend: pin anchor at current cursor-tip position
    for _, r in ipairs(session.cursors) do
      if not r._stopped then
        local info = vim.api.nvim_buf_get_extmark_by_id(r.buf, hl.ns, r.sel_mark_id, {})
        local anc_row, anc_col = info[1], info[2]
        -- Create anchor tracking extmark (right_gravity=false: won't drift on insert-before)
        r.anchor_mark_id = vim.api.nvim_buf_set_extmark(r.buf, hl.ns, anc_row, anc_col, {
          right_gravity = false,
          strict        = false,
        })
        r.mode = 'extend'
      end
    end
    session.extend_mode = true
  end
  hl.redraw(session)
end
```
