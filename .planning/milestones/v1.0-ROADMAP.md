## Milestone Summary

**Key Decisions:**

- Start fresh from master (VimL) — prior Lua branch used as reference only
- No backward compat with g:VM_xxx — setup() only, clean break
- Drop Python dependency — nvim_buf_get_offset replaces line2byte()
- Neovim 0.10+ minimum — use vim.iter, refined vim.bo/vim.wo, stable hl_mode on extmarks
- dofile() to load mini.test (dot in 'mini.test' dir name prevents require())
- debug.getinfo(1,'S').source for script path under nvim -l
- highlight.lua is Tier-0 — never requires region.lua (util lazy-required to prevent circularity)
- Read-then-clear-then-draw order in redraw() — positions cached before nvim_buf_clear_namespace
- undojoin between cursor iterations achieves single-undo-entry grouping for feedkeys-based edits
- GAP-01: define_groups() added as second line in M.setup()
- GAP-02: g_increment uses first/undojoin pattern identical to M.exec

**Issues Resolved:**
- BUG-02: scratch buffers undolevels=-1 — use nvim_create_buf(false, false)
- BUG-03/04: undojoin wrapped in pcall to guard invalid-context errors
- MISSING-01: VM_ groups not defined at startup — fixed by adding define_groups() to setup()
- MISSING-02: g_increment didn't use undojoin — N cursors created N undo entries, now 1

**Technical Debt Deferred to v2.0:**
- Full keymap table (all VM_* keys) deferred to Phase 6 — only 'v' key installed
- edit.lua exports implemented but not wired to user keymaps (Phase 6)
- config.get() not consumed by production code (Phase 5+)
- guicursor modification deferred to Phase 6
- Insert mode entry from _exec_change deferred to Phase 5
